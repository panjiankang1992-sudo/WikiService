# WikiService 离线版本部署方案

> 本文档描述如何将 WikiService 完全部署在内网环境中，不依赖任何外部服务。

---

## 一、系统架构总览

```
┌─────────────────────────────────────────────────────────┐
│                     用户浏览器                              │
│              http://localhost:23100                        │
└──────────────────┬──────────────────────────────────────┘
                   │ HTTP
┌──────────────────▼──────────────────────────────────────┐
│                   WikiService (Flask)                     │
│                   端口: 23100                            │
│  ┌─────────────┐ ┌──────────────┐ ┌──────────────────┐  │
│  │  知识查询    │ │  知识图谱     │ │   知识管理        │  │
│  │  /api/query │ │  /api/graph  │ │  /api/chunks     │  │
│  └──────┬──────┘ └──────┬───────┘ └────────┬─────────┘  │
│         │               │                  │            │
│  ┌──────▼───────────────▼──────────────────▼─────────┐  │
│  │              检索引擎 (Semantic + Keyword)            │  │
│  └──────────────────┬────────────────────────────────┘  │
└─────────────────────┼───────────────────────────────────┘
                      │ POST /api/embeddings
              ┌───────▼────────┐

              └────────────────┘
```

## 二、核心依赖

### 2.1 必需组件（全离线）

| 组件 | 版本 | 用途 | 离线说明 |
|------|------|------|----------|
| **Ollama** | ≥ 0.1.42 | 本地 Embedding 模型 | 无需网络 |
| **nomic-embed-text** | latest | 语义向量计算 | 首次 pull 后离线可用 |
| **Python** | ≥ 3.10 | 运行环境 | 系统自带或 conda |
| **Flask** | ≥ 3.0 | Web 服务 | pip install |
| **httpx** | ≥ 0.27 | HTTP 客户端 | pip install |
| **beautifulsoup4** | ≥ 4.12 | HTML 解析 | pip install |

### 2.2 可选组件

| 组件 | 用途 | 离线替代 |
|------|------|----------|
| DeepSeek API | 仅 LLM 评分（当前版本已移除） | 无需，搜索无需 LLM |
| 网络抓取 | URL 来源导入 | 使用 GitHub / 本地文件导入替代 |

## 三、部署步骤

### 3.1 第一步：安装 Ollama（仅首次）

```bash
# macOS
brew install ollama

# Linux
curl -fsSL https://ollama.com/install.sh | sh

# Windows: 下载安装包 https://ollama.com/download
```

### 3.2 第二步：下载 Embedding 模型（仅首次，需联网）

```bash
ollama pull nomic-embed-text
```

> 此步骤需要网络连接。模型下载后（约 274MB），后续运行完全离线。

**验证模型是否就绪：**
```bash
ollama list
# 输出应包含 nomic-embed-text
```

### 3.3 第三步：安装 Python 依赖

```bash
cd /path/to/WikiService/lightrag
pip install flask flask-cors httpx beautifulsoup4 lxml
```

或使用虚拟环境：
```bash
python -m venv .venv
.venv/bin/pip install flask flask-cors httpx beautifulsoup4 lxml
```

### 3.4 第四步：配置环境变量

```bash
# 必须设置（Ollama 方式，无需 API Key）
export EMBEDDING_PROVIDER=ollama
export EMBEDDING_MODEL=nomic-embed-text
export EMBEDDING_BASE_URL=http://localhost:11434

# 可选设置
export PORT=23100                           # 服务端口，默认 23100
export STORAGE_PATH=/opt/wikiservice/data  # 数据存储路径
export LOG_DIR=/opt/wikiservice/logs       # 日志路径
```

> **注意**：`DEEPSEEK_API_KEY` 在当前版本中仍需要设置一个占位值（任意字符串），但在纯搜索模式下不会被调用。建议设为占位值以兼容未来 LLM 评分功能。

```bash
export DEEPSEEK_API_KEY=offline-placeholder
```

### 3.5 第五步：启动服务

```bash
# 开发模式
python app.py

# 生产模式（后台运行）
nohup python app.py > /opt/wikiservice/logs/wikiservice_23100.log 2>&1 &
```

访问 `http://localhost:23100` 确认服务正常。

## 四、数据导入（离线）

### 4.1 本地文件目录导入

将本地 Markdown / 文本文件批量录入知识库：

```bash
curl -X POST http://localhost:23100/api/ingest/directory \
  -H "Content-Type: application/json" \
  -d '{"path": "/path/to/your/docs", "recursive": true}'
```

参数说明：
- `path`: 本地文件目录路径
- `recursive`: 是否递归扫描子目录（默认 true）

### 4.2 单文件上传

通过 Web 界面左侧「知识管理」→「添加知识」上传文件。

### 4.3 GitHub 仓库导入（需联网一次）

```bash
curl -X POST http://localhost:23100/api/ingest/github \
  -H "Content-Type: application/json" \
  -d '{"repo": "owner/repo", "branch": "main"}'
```

> 首次导入需要联网拉取 GitHub 文件内容。导入完成后数据保存在本地，后续查询无需网络。

### 4.4 手动文本输入

通过 Web 界面「知识管理」→「手动输入」添加单条知识。

## 五、数据存储结构

```
STORAGE_PATH/
├── text_chunks.json       # 所有文档 chunks（含文本、来源、元信息）
├── embeddings.json         # 向量缓存（key: chunk_id, value: {vec, _hash}）
└── sources/               # 来源配置文件（可忽略）
```

**文件格式：**

`text_chunks.json`:
```json
[
  {
    "id": "chunk-xxx",
    "source": "github:owner/repo",
    "metadata": {
      "path": "README.md",
      "directory": "/docs"
    },
    "text": "# 文档标题\n正文内容...",
    "created_at": "2026-05-15T10:00:00"
  }
]
```

`embeddings.json`:
```json
{
  "chunk-xxx": {
    "vec": [0.123, -0.456, ...],
    "_hash": "a1b2c3d4"
  }
}
```

## 六、搜索原理详解

### 6.1 检索流程

```
用户查询
  │
  ├─ ① 语义搜索（Ollama Embedding）
  │    计算查询向量 → 与所有 chunks 余弦相似度 → 排序
  │
  └─ ② 关键词 Fallback（BM25）
       中文 bigram + 英文词边界匹配
       仅在语义搜索无结果时触发
```

### 6.2 综合关联度排序

每个文档计算以下分数：

```
综合关联度 = 语义相似度 × 0.55 + 关键词匹配强度 × 0.45
```

**关键词匹配强度（0-1）：**

| 条件 | 强度 |
|------|------|
| 标题词组精确命中 | 0.9 |
| 英文单词命中（词边界，≥3字符） | 0.8 |
| 标题 bigram ≥ 3 个命中 | 0.65 |
| 标题 bigram = 2 个命中 | 0.5 |
| 标题 bigram = 1 个命中 | 0.3 |
| 正文 bigram 命中（标题无关键词时） | 0.15 |

**结果分类阈值：**

| 分类 | 条件 |
|------|------|
| HIGH（推荐文档） | 关键词命中 + 语义 ≥ 50% |
| MEDIUM A（可能相关） | 关键词命中 + 语义 ≥ 30% |
| MEDIUM B（可能相关） | 极短英文查询（≤4字符）+ 语义 ≥ 40% + 正文命中 |
| MEDIUM C（可能相关） | 正文命中（首 bigram）+ 语义 ≥ 50%（标题无关键词时） |

所有结果按综合关联度统一排序返回。

### 6.3 去重规则

相同 `source + title` 的文档只保留一条（取最高分）。

## 七、知识图谱

图谱自动从已导入文档中提取：
- **实体**：技术名词、中文短语、代码片段
- **关系**：同一文档中的实体相互关联
- **可视化**：D3.js 力导向图，支持缩放、节点点击、关联高亮

## 九、一键部署脚本

```bash
#!/bin/bash
# offline-deploy.sh — 单机离线部署脚本

set -e

# ========== 配置 ==========
OLLAMA_URL="${OLLAMA_URL:-http://localhost:11434}"
SERVICE_PORT="${SERVICE_PORT:-23100}"
STORAGE="${STORAGE:-./data}"
LOG_DIR="${LOG_DIR:-./logs}"
DEEPSEEK_KEY="${DEEPSEEK_KEY:-offline-placeholder}"

# ========== 检查 Ollama ==========
echo "[1/4] 检查 Ollama..."
if ! curl -s "$OLLAMA_URL/api/tags" > /dev/null 2>&1; then
    echo "ERROR: Ollama 未运行，请先启动: ollama serve"
    exit 1
fi

echo "[2/4] 检查模型..."
if ! ollama list | grep -q nomic-embed-text; then
    echo "WARNING: nomic-embed-text 未安装，首次需要联网下载："
    echo "  ollama pull nomic-embed-text"
fi

# ========== 安装依赖 ==========
echo "[3/4] 安装 Python 依赖..."
pip install flask flask-cors httpx beautifulsoup4 lxml -q

# ========== 启动服务 ==========
echo "[4/4] 启动 WikiService..."
mkdir -p "$STORAGE" "$LOG_DIR"

export PORT="$SERVICE_PORT"
export STORAGE_PATH="$STORAGE"
export LOG_DIR="$LOG_DIR"
export DEEPSEEK_API_KEY="$DEEPSEEK_KEY"
export EMBEDDING_PROVIDER=ollama
export EMBEDDING_MODEL=nomic-embed-text
export EMBEDDING_BASE_URL="$OLLAMA_URL"

nohup python app.py > "$LOG_DIR/wikiservice_${SERVICE_PORT}.log" 2>&1 &

sleep 2
if curl -s "http://localhost:${SERVICE_PORT}/health" | grep -q ok; then
    echo "✓ WikiService 已启动: http://localhost:${SERVICE_PORT}"
else
    echo "ERROR: 服务启动失败，查看日志: $LOG_DIR/wikiservice_${SERVICE_PORT}.log"
fi
```

## 十、故障排查

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| `EMBEDDING_PROVIDER environment variable is required` | 旧版本代码 | 同步最新 app.py |
| 搜索返回 0 结果 | 知识库为空 | 先通过「知识管理」导入文档 |
| Ollama 超时 | 模型未加载 | `ollama run nomic-embed-text` 预热 |
| 端口已被占用 | 多个实例 | `kill $(lsof -ti TCP:23100)` 或改用其他端口 |
| 语义分数过低 | chunk 过多 | 检查文档是否被正确分块 |
